### Exercise 11.1: Describe the differences between a map and a vector.
**原题：** Describe the differences between a map and a vector.

**你的答案：**
```
map的结构首先和vector就不一样
vector是顺序结构， 一个下标对应一个元素， 下标访问一定是通过一个size_type的数字

然而map不一样，map内部是由一个一个pair模版构成的每个模版里面又有两个成员对象， 因此访问map的元素就不一定是一个size_type的下标了， 要看这个map的模版定义，也就是key的类型
```

**批改结果：** ⭐⭐⭐ 3/5
- **概念理解：** 3/5 - 基本理解了两种容器的核心差异，但表述不够精确
- **代码质量：** 不适用（本题为概念描述题）
- **逻辑完整：** 2.5/5 - 遗漏了多个重要差异点
- **表达清晰：** 3/5 - 思路清晰但用词可以更专业

**详细分析：**
- ✅ 正确指出了vector是顺序结构，使用数字下标访问
- ✅ 正确指出map由pair组成，key决定访问方式
- ⚠️ 描述中存在用词不准确问题："pair模版构成的每个模版"应为"pair对象组成"
- ⚠️ 遗漏了多个重要差异：存储方式（连续内存 vs 树结构）、排序特性、查找效率等

**正确答案：**
```cpp
// 概念描述（非代码）：
1. **底层数据结构**：vector使用连续内存数组，map通常使用红黑树（有序）或哈希表（无序）
2. **元素访问方式**：vector通过整数索引（0-based）访问；map通过键（key）访问
3. **元素特性**：vector存储单一类型元素；map存储键值对（key-value pairs）
4. **排序特性**：vector保持插入顺序；map默认按键排序（std::map）
5. **查找效率**：vector中查找为O(n)；map中查找为O(log n)（有序）或O(1)（无序）
6. **插入位置**：vector主要在末尾高效插入；map在任何位置插入都高效
7. **内存布局**：vector元素连续存储；map元素分散存储（通过指针连接）
```

### Exercise 11.2: Give an example of when each of list, vector, deque, map, and set might be most useful.
**原题：** Give an example of when each of list, vector, deque, map, and set might be most useful.

**你的答案：**
```
查看是否在人员名单， 因为只需要查找是否在本集合: set 
查找单词出现频率， 特定单词对应频率，符合pair特征: map
字典序存储出现过的一些列单词出现的字母：符合低成本中间大量插入的特征：list
人口普查的年龄数据存储，符合vector适合存储连续大量的数据特点：vector
餐厅排队点号， 因为这个只需要在头尾解析操作：deque
```

**批改结果：** ⭐⭐⭐ 3/5
- **概念理解：** 3.5/5 - 大部分例子选择合理，但部分解释需要优化
- **代码质量：** 不适用（本题为概念描述题）
- **逻辑完整：** 3/5 - 例子基本正确，但"字典序存储"的解释不够准确
- **表达清晰：** 3/5 - 表达基本清晰，但部分描述可以更精确

**详细分析：**
- ✅ **set示例**：人员名单检查（需要快速查找是否存在） - 选择正确
- ✅ **map示例**：单词频率统计（键值对存储） - 选择正确
- ⚠️ **list示例**："字典序存储出现过的一系列单词出现的字母" - 描述混乱且不准确
- ✅ **vector示例**：人口普查年龄数据（大量数据，顺序访问） - 选择合理
- ✅ **deque示例**：餐厅排队（双端操作） - 选择合理

**正确答案：**
```cpp
// 各容器适用场景示例：
1. **vector**：存储学生成绩列表，需要随机访问任意学生的成绩
   // 优点：连续存储，缓存友好，支持快速随机访问

2. **list**：维护一个待处理任务列表，需要频繁在任意位置插入和删除
   // 优点：任意位置插入删除O(1)，不支持随机访问

3. **deque**：实现一个撤销操作历史记录，支持两端添加和删除
   // 优点：两端高效插入删除，支持随机访问（但比vector稍慢）

4. **map**：实现电话簿，通过姓名查找电话号码
   // 优点：按键排序，查找效率高，键值对存储

5. **set**：检查拼写错误的单词，快速判断单词是否在字典中
   // 优点：自动去重，快速查找，元素唯一
```
