# 📘 C++ 输入输出流（I/O Streams）

> 来源说明：C++ Primer 输入输出流章节 | 本节涵盖：流类层次结构、文件操作、字符串流、状态管理等核心概念

---

### 📚 C++ I/O 流核心概念知识表

| 类别 | 概念 | 定义 | 关键点 |
|------|------|------|--------|
| **🔧 流类基础** | **iostream** | 处理控制台输入输出的库类 | • 基础控制台I/O类<br>• 其他流类的基类 |
|  | **fstream** | 处理命名文件输入输出的库类 | • 文件导向的I/O操作<br>• 包含open/close成员函数 |
|  | **stringstream** | 处理内存字符串输入输出的库类 | • 基于内存的I/O操作<br>• 使用字符串缓冲区 |
| **🔄 继承体系** | **继承关系** | 类型从其他类型继承接口的编程特性 | • ifstream/istringstream ← istream<br>• ofstream/ostringstream ← ostream<br>• 支持多态使用 |
|  | **istream** | 输入流基类 | • 定义通用输入操作<br>• 被ifstream和istringstream继承 |
|  | **ostream** | 输出流基类 | • 定义通用输出操作<br>• 被ofstream和ostringstream继承 |
| **📁 文件流操作** | **文件流** | 读写命名文件的流对象 | • 额外提供open和close成员<br>• 打开新文件前必须先关闭当前文件 |
|  | **ifstream** | 读取输入文件的文件流 | • 默认打开模式：in<br>• 继承自istream |
|  | **ofstream** | 写入输出文件的文件流 | • 默认打开模式：out<br>• 继承自ostream |
|  | **fstream** | 读写同一文件的文件流 | • 默认模式：in & out<br>• 支持双向文件访问 |
|  | **文件模式** | fstream类定义的标志，控制文件使用方式 | • 在打开文件时指定<br>• 控制读写权限和行为 |
| **📝 字符串流操作** | **字符串流** | 读写字符串的流对象 | • 额外提供str成员函数<br>• 可获取/设置底层字符串 |
|  | **istringstream** | 从给定字符串读取的字符串流 | • 对字符串执行输入操作<br>• 继承自istream |
|  | **ostringstream** | 写入给定字符串的字符串流 | • 向字符串执行输出操作<br>• 继承自ostream |
|  | **stringstream** | 读写给定字符串的字符串流 | • 支持双向字符串操作<br>• 完整的字符串I/O能力 |
| **⚙️ 状态管理** | **条件状态** | 流类使用的标志和相关函数，指示流是否可用 | • 跟踪流健康状况（错误、EOF等）<br>• 错误修复后才能继续使用 |
|  | **str成员函数** | 字符串流的重载成员函数 | • 无参数调用：返回关联字符串<br>• 带字符串参数：关联到该字符串副本 |
