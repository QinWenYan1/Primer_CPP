
## 📘 章节笔记：标准库算法与迭代器

| 类别 | 概念 | 定义 | 关键点 |
| :--- | :--- | :--- | :--- |
| **🔄 迭代器类别** | **输入迭代器 (input iterator)** | 可以读取，但不能写入序列元素的迭代器。 | • 用于只读算法。<br>• 支持 `++` 和 `*`（解引用读取）。 |
| | **输出迭代器 (output iterator)** | 可以写入，但不一定能读取序列元素的迭代器。 | • 用于只写算法。<br>• 支持 `++` 和 `*`（解引用赋值）。 |
| | **前向迭代器 (forward iterator)** | 可以读写元素，但不支持 `--` 操作（即只能单向移动）。 | • 继承输入/输出迭代器的所有功能。<br>• 可多次遍历同一序列。 |
| | **双向迭代器 (bidirectional iterator)** | 在前向迭代器基础上，增加了使用 `--` 向后移动的能力。 | • 支持正向和反向遍历。<br>• 如 `list`, `set`, `map` 的迭代器。 |
| | **随机访问迭代器 (random-access iterator)** | 在双向迭代器基础上，支持与整数的加减运算、关系比较和下标运算符。 | • 支持任意位置的快速跳转。<br>• 如 `vector`, `deque`, 内置数组的指针。 |
| **📊 算法特性** | **泛型算法 (generic algorithms)** | 标准库中约100个与类型无关的算法，通过迭代器操作序列。 | • 不直接操作容器，只操作迭代器。<br>• 通过迭代器类别要求来规定其功能。 |
| | **算法不改变容器大小** | 算法本身永远不会直接添加或删除容器中的元素。 | • 只能复制、移动或重排元素。<br>• **例外**：使用插入迭代器可以间接添加元素。 |
| **🔗 特殊迭代器（适配器）** | **插入迭代器 (insert iterator)** | 一种迭代器适配器，绑定到容器，赋值操作会向容器**添加**元素。 | • 是算法能够“改变”容器大小的关键机制。 |
| | **back_inserter** | 插入迭代器适配器，使用容器的 `push_back` 方法在末尾添加元素。 | • 适用于支持 `push_back` 的容器（如 `vector`, `deque`, `list`）。 |
| | **front_inserter** | 插入迭代器适配器，使用容器的 `push_front` 方法在开头添加元素。 | • 适用于支持 `push_front` 的容器（如 `deque`, `list`）。 |
| | **inserter** | 插入迭代器适配器，使用容器的 `insert` 方法在给定迭代器**之前**插入元素。 | • 适用于所有支持 `insert` 的容器。 |
| | **流迭代器 (stream iterator)** | 可绑定到输入/输出流的迭代器。 | • `istream_iterator`: 读取输入流。<br>• `ostream_iterator`: 写入输出流。 |
| | **反向迭代器 (reverse iterator)** | 一种迭代器适配器，移动方向与底层迭代器相反（`++` 向后，`--` 向前）。 | • 用于反向遍历容器。 |
| | **移动迭代器 (move iterator)** | 迭代器适配器，解引用时返回右值引用，用于移动元素而非复制。 | • 在需要移动语义优化时使用。 |
| **🎯 函数对象与谓词** | **可调用对象 (callable object)** | 可以作为函数调用运算符左操作数的对象。 | • 包括：函数指针、lambda表达式、重载了 `operator()` 的类对象。 |
| | **谓词 (predicate)** | 返回可转换为 `bool` 的类型的函数，常被算法用于元素测试。 | • **一元谓词**：接受一个参数。<br>• **二元谓词**：接受两个参数。 |
| | **lambda表达式** | 一种匿名的、内联的可调用代码单元。 | • 结构：`[捕获列表] (参数列表) -> 返回类型 { 函数体 }`。<br>• 可省略返回类型（单条return语句时可推断）。<br>• 捕获列表允许访问外层函数的变量。 |
| | **bind** | 标准库函数，用于将可调用对象的某些参数绑定到固定值。 | • 位于 `<functional>` 头文件。<br>• 用于生成新的可调用对象。 |
| | **ref / cref** | 生成一个可拷贝的对象，分别持有对非常量/常量对象的引用。 | • 用于将引用传递给 `bind` 等接受值语义的函数。 |
| **⚠️ 容器特定行为** | **list 和 forward_list 的专用算法** | 这些容器定义了自身版本的一些通用算法（如 `sort`, `merge`, `remove`）。 | • 与泛型算法不同，这些专用版本会**修改容器本身**。<br>• 性能通常更优，因为利用了链表的结构特性。 |
