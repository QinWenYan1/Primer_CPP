### 📘 章节总结：运算符重载与类型转换

> 本章主要介绍了C++中运算符重载的规则、特殊运算符的要求、函数对象的概念以及类类型转换的定义和使用。

| 类别 | 概念 | 定义 | 关键点 |
| :--- | :--- | :--- | :--- |
| **⚙️ 基础规则** | **重载运算符** | 重新定义内置运算符含义的函数，名为 `operator` 后跟运算符符号。 | • 至少有一个操作数是类类型<br>• 保持内置运算符的优先级、结合性和操作数个数<br>• 可作为成员函数或非成员函数定义 |
| | **作为成员函数的重载运算符** | 将运算符定义为类的成员函数。 | • 隐式的 `this` 指针绑定到第一个操作数 |
| **🔑 特殊运算符** | **必须为类成员的运算符** | 赋值（`=`）、下标（`[]`）、函数调用（`()`）和箭头（`->`）运算符。 | • 必须定义为类的成员函数，不能是普通非成员函数 |
| **📦 函数对象** | **函数对象** | 重载了函数调用运算符 `operator()` 的类的对象。 | • 可以像函数一样被调用<br>• 常与标准算法结合使用 |
| | **函数表** | 存储可调用对象的容器，通常是 `map` 或 `vector`。 | • 用于实现回调、策略模式等 |
| | **函数模板** | 标准库模板，可以代表任何可调用类型。 | • 例如 `std::function` |
| | **调用签名** | 可调用对象的接口，包括返回类型和括号括起来的、逗号分隔的参数类型列表。 | • 例如 `int (int, int)` 表示一个接受两个 `int` 并返回 `int` 的可调用对象 |
| | **λ表达式** | 定义简单函数对象类的简洁方式。 | • 编译器会将其转化为一个匿名函数对象 |
| **🔄 类型转换** | **类类型转换 / 用户定义转换** | 通过构造函数和转换运算符定义到/从类类型的转换。 | • 由编译器自动执行（隐式转换）或强制执行（显式转换） |
| | **转换构造函数** | 可以被单个参数调用的非`explicit`构造函数。 | • 定义了从参数类型到类类型的转换 |
| | **转换运算符** | 定义从类类型到其他类型的转换的成员函数。 | • 必须是类成员，通常是 `const`<br>• 无返回类型，无参数<br>• `operator int()` 返回 `int`，`operator string()` 返回 `string`，以此类推 |
| | **显式转换运算符** | 由 `explicit` 关键字修饰的转换运算符。 | • 禁止在隐式转换中使用<br>• 仅在条件表达式（如 `if`、`while`）中允许隐式使用 |
