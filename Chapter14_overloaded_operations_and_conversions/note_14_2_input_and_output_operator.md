<a id="id1"></a>
## ✅ 知识点1: 重载输出运算符`<<`

**理论**
* **输出运算符(output operator)的第一个参数**：
    * 通常是一个**非const**(nonconst)的`ostream`对象的引用。
    * `ostream`对象必须是非const，因为写入流会改变其状态；
    * 使用引用是因为**不能拷贝`ostream`对象**。
* **第二个参数**：
    * 通常是一个**const引用**，类型为要打印的类类型。
    * 使用引用是为了避免拷贝参数；
    * 使用const是因为打印对象通常不会改变该对象。
* **返回值**：为了与其他输出运算符保持一致，`operator<<`通常返回它的`ostream`参数（即第一个参数）。

<a id="id2"></a>
## ✅ 知识点2: `Sales_data`的输出运算符 

**理论**
* 以下是为`Sales_data`类编写的输出运算符，它打印三个数据成员和计算出的平均售价。
* 每个元素之间用空格分隔，最后返回写入的`ostream`引用。

**注意点**
* 除了函数名，这个定义与之前的`print`函数（§7.1.3）完全相同。
* 打印完所有值后，运算符返回它刚刚写入的`ostream`引用。

**教材示例代码**
```cpp
ostream &operator<<(ostream &os, const Sales_data &item)
{
    os << item.isbn() << " " << item.units_sold << " "
       << item.revenue << " " << item.avg_price();
    return os;
}
```

<a id="id3"></a>
## ✅ 知识点3: 输出运算符通常做最小格式化 

**理论**
* 内置类型的输出运算符几乎不做格式化，尤其**不打印换行符**。
* 用户期望类类型的输出运算符也遵循相同的行为。如果运算符打印了换行，用户就无法在同一行中与对象一起打印描述性文本。
* 只做最小格式化的输出运算符让用户能够自行控制输出的具体格式。

**最佳实践**
> 一般地，输出运算符应该只打印对象的内容，做**最小格式化**，并且**不应打印换行符**。

<a id="id4"></a>
## ✅ 知识点4: IO运算符必须是非成员函数

**理论**
* 符合iostream库惯例的输入和输出运算符必须是普通的**非成员函数**。
    * 它们不能是类的成员，否则左操作数将是该类类型的对象（例如 `data << cout;`），但我们需要左操作数是流对象。
* 如果这些运算符是任何类的成员，那它们必须是`istream`或`ostream`的成员，然而这些类是标准库的一部分，我们**无法向标准库类添加成员**。
* 因此，为自定义类型定义IO运算符时，必须定义为非成员函数。
* 通常IO运算符需要读写类的非公有数据成员，因此它们通常需要被声明为**友元**(`friend`)（§7.2.1）。

<a id="id5"></a>
## ✅ 知识点5: 重载输入运算符`>>`

**理论**
* **输入运算符(input operator)的第一个参数**：一个引用，指向要读取的流（通常是`istream`）。
* **第二个参数**：一个引用，指向要读入数据的（**非const**）对象。
* **返回值**：通常返回对给定流的引用。
* 第二个参数必须是非const，因为输入运算符的目的是将数据读入该对象。

---

<a id="id6"></a>
## ✅ 知识点6: `Sales_data`的输入运算符 

**理论**
* 下面是`Sales_data`的输入运算符示例。它先读取三个值，然后检查读取是否成功，成功则计算`revenue`，失败则将对象重置为默认状态。

**教材示例代码**
```cpp
istream &operator>>(istream &is, Sales_data &item)
{
    double price; // 无需初始化，我们将在使用前读入price
    is >> item.bookNo >> item.units_sold >> price;
    if (is)                     // 检查输入是否成功
        item.revenue = item.units_sold * price;
    else
        item = Sales_data();    // 输入失败：将对象置为默认状态
    return is;
}
```

**代码解析**
* 除了`if`语句，这个定义与之前的`read`函数（§7.1.3）相似。
* `if`检查读取是否成功。如果发生IO错误，运算符将给定对象重置为空`Sales_data`（通过赋予一个默认初始化的`Sales_data`对象）。这样确保了对象处于一致的状态。

<a id="id7"></a>
## ✅ 知识点7: 输入运算符必须处理输入失败的可能性

> ⚠️ **注意**：输入运算符必须处理输入可能失败的情况；输出运算符通常不用关心。

---

<a id="id8"></a>
## ✅ 知识点8: 输入时的错误 

**理论**
* 输入运算符可能遇到的错误包括：
  * **读取操作失败**：例如流中包含错误类型的数据。在读取`bookNo`之后，输入运算符期望接下来两个项是数值数据。如果输入了非数值数据，那次读取及后续对流的使用都会失败。
  * **遇到文件结束符(end-of-file)** 或流上的其他错误。

---

<a id="id9"></a>
## ✅ 知识点9: 检查输入成功并重置对象 

**理论**
* 我们不在每次读取后立即检查，而是在读取完所有数据、使用这些数据之前进行一次检查：

**教材示例代码**
```cpp
if (is)    // 检查输入是否成功
    item.revenue = item.units_sold * price;
else
    item = Sales_data(); // 输入失败：将对象置为默认状态
```

**代码解析**

* 如果任何读取操作失败，`price`将是未定义的值。因此在使用`price`之前，我们先检查输入流是否仍然有效。如果有效，计算并存储结果到`revenue`；如果出错，我们不去追究是哪个输入失败，而是将整个对象重置为空`Sales_data`。
* 重置后，`item`的`bookNo`成员为空字符串，`revenue`和`units_sold`为零。
* **将对象置于有效状态**尤其重要，如果错误发生前对象可能已被部分改变。例如，在输入运算符中，可能在成功读取新的`bookNo`后发生错误。如果错误发生在读取`units_sold`和`revenue`之前，旧对象的这两个成员将保持不变，导致`bookNo`与那些旧数据关联。
* 通过将对象留在有效状态，可以在一定程度上保护忽略输入错误的用户：对象处于可用状态（所有成员都已定义），且数据内部一致，不会产生误导性结果。

<a id="id11"></a>
## ✅ 知识点11: 指示错误 

**理论**
* 有些输入运算符需要额外的数据验证。例如，输入运算符可能会检查读取的`bookNo`是否符合特定格式。
* 在这种情况下，即使实际的IO操作技术上成功，输入运算符也可能需要设置流的条件状态以指示失败（§8.1.2）。
* 通常，输入运算符只应设置`failbit`。设置`eofbit`意味着文件已耗尽，设置`badbit`表示流已损坏。这些错误最好留给IO库本身去指示。

---

## 🔑 核心要点总结
1. **输出运算符**：第一个参数为`nonconst ostream&`，第二个参数为`const`引用到类类型，返回`ostream&`。必须为非成员函数，通常声明为友元。应做最小格式化，不打印换行。
2. **输入运算符**：第一个参数为`istream&`，第二个参数为**非const**引用到类类型，返回`istream&`。必须处理输入失败的可能，并在失败时将对象置于有效状态（如默认状态）。
3. **错误处理**：输入运算符应检查读取后的流状态，若失败则重置对象。对于格式验证错误，可以设置`failbit`。
4. **非成员函数**：IO运算符不能是类的成员，因为左操作数必须是流对象，且不能修改标准库类。

## 📌 考试速记版
* **输出运算符签名**：`ostream& operator<<(ostream&, const Class&);`
* **输入运算符签名**：`istream& operator>>(istream&, Class&);`
* **必须是非成员**：左操作数是流，不能是类对象。
* **错误处理要点**：输入后检查`if(is)`，失败则重置对象（`obj = Class()`），通常只设置`failbit`。
* **格式化原则**：输出最小格式化，不打印换行；输入需处理错误。

**记忆口诀**：
> 输出参数常引用，const对象不用愁；  
> 返回流引用是惯例，最小格式不换行。  
> 输入对象非常量，错误处理记心上；  
> 失败重置保一致，非成员友元来帮忙。
