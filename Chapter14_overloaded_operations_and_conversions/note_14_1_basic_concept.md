<a id="id1"></a>
## ✅ 知识点1: 重载运算符的基本定义

**理论**
* 重载运算符(overloaded operators)是**具有特殊名称的函数**：关键字 `operator` 后跟要定义的运算符符号。
* 与任何其他函数一样，重载运算符有**返回类型(return type)**、**参数列表(parameter list)**和**函数体(body)**。

---

<a id="id2"></a>
## ✅ 知识点2: 重载运算符的参数数量规则

**理论**
* 重载运算符函数的**参数数量与运算符操作数(operands)的数量相同**。
* 一元运算符(unary operator)有一个参数；二元运算符(binary operator)有两个参数。
* 在二元运算符中，**左操作数传递给第一个参数，右操作数传递给第二个参数**。
* **除函数调用运算符 `operator()` 外**，重载运算符**不能有默认参数(default arguments)**。

---

<a id="id3"></a>
## ✅ 知识点3: 作为成员函数时的参数规则

**理论**
* 如果运算符函数是**成员函数(member function)**，则第一个（左）操作数隐式绑定到 `this` 指针。
* 因此，成员运算符函数的**显式参数数量比操作数数量少一个**（因为左操作数由 `this` 提供）。

---

<a id="id4"></a>
## ✅ 知识点4: Note：成员运算符函数的隐式this绑定

**理论**
> **Note**
> * 当重载运算符为成员函数时，`this` 绑定到左操作数。
> * 成员运算符函数的显式参数数量比操作数数量少一个。

---

<a id="id5"></a>
## ✅ 知识点5: 运算符函数的参数必须包含至少一个类类型

**理论**
* 运算符函数**必须是类的成员，或至少有一个参数为类类型(class type)**。
* **不能重新定义应用于内置类型操作数的运算符含义**。

**教材示例代码**
```cpp
// error: cannot redefine the built-in operator for ints
int operator+(int, int);
```

**注意点**
* ⚠️ **核心限制**：此限制确保了程序员**无法改变内置运算符的原有行为**。

<a id="id7"></a>
## ✅ 知识点7: 不能发明新运算符符号

**理论**
* **只能重载已有的运算符**，不能发明新的运算符符号。
* 例如，不能定义 `operator**` 来表示幂运算。

---

<a id="id8"></a>
## ✅ 知识点8: 一元/二元符号的复用

**理论**
* 四个符号 `+`、`-`、`*` 和 `&` 既可以作为一元运算符也可以作为二元运算符。
* 这些运算符可以被重载为一元或二元版本，**参数的数量决定了定义的是哪个运算符**。

---

<a id="id9"></a>
## ✅ 知识点9: 重载运算符的优先级与结合性不变

**理论**
* 重载运算符与对应的内置运算符具有相同的**优先级(precedence)**和**结合性(associativity)**。
* 无论操作数的类型如何，表达式 `x = y + z;` 总是等价于 `x = (y + z);`。

---

<a id="id10"></a>
## ✅ 知识点10: 直接调用重载运算符函数

**理论**
* 通常我们通过使用运算符于适当类型的参数上间接“调用”重载运算符。
* 但也可以像调用普通函数一样**直接调用重载运算符函数**：使用函数名并传递适当数量和类型的参数。

**教材示例代码**
```cpp
// equivalent calls to a nonmember operator function
data1 + data2;                // normal expression
operator+(data1, data2);       // equivalent function call
```
* 上面两个调用等价，都调用非成员函数 `operator+`，传递 `data1` 作为第一个参数，`data2` 作为第二个。

* 对于成员运算符函数，可以通过对象（或指针）使用点（或箭头）运算符显式调用：
```cpp
data1 += data2;                // expression-based "call"
data1.operator+=(data2);       // equivalent call to a member operator function
```
* 两条语句都调用成员函数 `operator+=`，将 `this` 绑定到 `data1` 的地址，并传递 `data2` 作为参数。

<a id="id11"></a>
## ✅ 知识点11: 不应重载的运算符

**理论**
* 内置运算符中，有些保证了操作数的求值顺序（如逻辑与 `&&`、逻辑或 `||`、逗号运算符 `,`）。
* 但重载运算符本质上是函数调用，**这些保证不再适用**。
  * 重载的 `&&` 或 `||` **不保留短路求值(short-circuit evaluation)** 属性：两个操作数总是会被求值。
* 由于重载版本不保留求值顺序和/或短路求值，**通常重载它们是糟糕的想法**，用户会因期望的保证未实现而感到惊讶。
* 逗号运算符和取地址运算符 `&` 也有内置含义，**通常不应重载**，否则会使用户感到意外。

---

<a id="id12"></a>
## ✅ 知识点12: Best Practices：避免重载逗号、取地址、逻辑与、逻辑或

**理论**
> **Best Practices**
> * 通常，**逗号(comma)、取地址(address-of)、逻辑与(logical AND)和逻辑或(logical OR)运算符不应被重载**。

---

<a id="id13"></a>
## ✅ 知识点13: 定义应与内置含义一致

**理论**
* 设计类时，应先考虑需要哪些操作，再决定将这些操作定义为普通函数还是重载运算符。
* **与运算符有逻辑映射的操作是重载运算符的好候选**：
  * 如果类执行IO，则定义移位运算符(`<<`、`>>`)以与内置类型的IO行为一致。
  * 如果类有相等性测试操作，定义 `operator==`。如果定义了 `==`，通常也应定义 `operator!=`。
  * 如果类有单一自然的排序操作，定义 `operator<`。如果定义了 `<`，通常应提供所有关系运算符。
* **重载运算符的返回类型应与内置版本兼容**：
  * 逻辑和关系运算符应返回 `bool`。
  * 算术运算符应返回类类型的值。
  * 赋值和复合赋值应返回**左操作数的引用**。

---

<a id="id14"></a>
## ✅ 知识点14: 赋值与复合赋值运算符的行为

**理论**
* 赋值运算符(`=`)的行为应与合成版本类似：赋值后，左右操作数的值相同，且**返回左操作数的引用**。
* 重载的赋值应**推广内置赋值的含义，而不是绕过它**。

---

<a id="id15"></a>
## ✅ 知识点15: 谨慎使用运算符重载（Caution）

**理论**
> **Caution: Use Operator Overloading Judiciously**
* 每个运算符都与内置类型的特定含义相关联（如 `+` 表示加法）。
* 将运算符映射到类类型的类似操作可以提供便捷的简写。例如，`string` 使用 `+` 表示连接。
* **运算符重载最有用的场景是存在逻辑映射**，使用重载运算符可以使程序更自然、直观。
* **过度或滥用运算符重载会使类难以理解**。
  * 明显的滥用很少见（如用 `+` 做减法）。
  * 更常见但不推荐的是**强行将运算符的“正常”含义扭曲以适应类型**。运算符应只用于对用户而言无歧义的操作。如果运算符可能有多种解释，则含义模糊。
* 如果类有算术或位运算符，通常也**应提供对应的复合赋值运算符**（如 `+=` 应与 `+` 后跟 `=` 的行为一致）。

<a id="id16"></a>
## ✅ 知识点16: 选择成员或非成员实现的指南

**理论**
* 定义重载运算符时，必须决定是将其设为类的成员还是普通的非成员函数。
* 有些运算符**必须定义为成员**：
  * 赋值(`=`)、下标(`[]`)、调用(`()`)、成员访问箭头(`->`)。
* 复合赋值运算符通常应为成员，但不是必须的。
* 改变对象状态或与给定类型紧密相关的运算符（如自增、自减、解引用）通常应为成员。
* **对称运算符(symmetric operators)**（可能转换任一操作数，如算术、相等、关系、位运算符）通常应定义为普通的非成员函数。
* 原因：若定义为成员，则左操作数必须是该类类型的对象，无法实现混合类型表达式（如 `"hi" + s`）。作为非成员，允许对任一操作数进行类型转换，只要至少有一个操作数是类类型且可转换。

**教材示例代码**
```cpp
string s = "world";
string t = s + "!";   // ok: 可以将 const char* 加到 string
string u = "hi" + s;  // 如果 + 是 string 的成员，则会出错
```
* 如果 `+` 是 `string` 的成员，`"hi" + s` 等价于 `"hi".operator+(s)`，而 `"hi"` 是 `const char*`，没有成员函数。
* 因为 `string` 将 `+` 定义为非成员，`"hi" + s` 等价于 `operator+("hi", s)`，允许将 `"hi"` 转换为 `string`。

**注意点**
* ⚠️ **必须为成员的运算符**：`=`、`[]`、`()`、`->`。
* ⚠️ **建议为成员**：复合赋值、自增、自减、解引用。
* ⚠️ **建议为非成员**：算术、相等、关系、位运算符等对称运算符。

---

## 🔑 核心要点总结
1. **重载运算符是特殊函数**：使用 `operator` 关键字定义，参数数量与操作数一致。成员函数通过 `this` 隐式绑定左操作数，显式参数少一个。
2. **必须至少一个类类型参数**，不能改变内置类型的运算符含义。可重载大部分运算符，但 `::`、`.*`、`.`、`?:` 不能重载。
3. **重载运算符的优先级和结合性不变**，但短路求值、顺序保证会丢失（如 `&&`、`||`、逗号），因此通常不应重载这些运算符。
4. **设计应一致**：重载运算符应反映内置含义（返回类型、副作用）。赋值返回左值引用，算术返回新对象，关系返回 `bool`。
5. **成员/非成员选择**：`=`、`[]`、`()`、`->` 必须为成员；复合赋值、自增减通常为成员；对称运算符通常为非成员以支持混合类型。

## 📌 考试速记版
* **重载运算符语法**：`返回类型 operator符号(参数列表)`
* **成员函数参数数** = 操作数 - 1（因为this）
* **必须至少一个类类型参数**，不能全为内置类型
* **不能重载的运算符**：`::` `.*` `.` `?:`
* **不应重载的运算符**：`&&` `||` `,` `&`（逗号和取地址）
* **必须为成员的运算符**：`=` `[]` `()` `->`
* **建议为成员**：`+=` `-=` 等复合赋值、`++` `--` `*`（解引用）
* **建议为非成员**：`+` `-` `*` `/` `==` `<` `>` 等对称运算符
* **赋值运算符应返回**：`类类型&`（左操作数引用）
