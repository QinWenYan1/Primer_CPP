**你的答案：**  
+=， +， -， -= 这几个操作符应该支持， 其他的运算符号就不应该支持了， 因为对于一个表示了ISBN, units_sold, 以及price的类， 给他们%, *符号在逻辑上并不符合，走不通

**批改结果：** ⭐⭐ (2/5)  
- **概念理解：** ✅ 你理解了算术运算符应该与类的逻辑语义相匹配，这是一个正确的思路。但你对 `Sales_data` 类的常用操作理解不够全面：通常 `Sales_data` 只支持 `+` 和 `+=` 用于合并销售记录，而 `-` 和 `-=` 在商业逻辑上很少有意义（除非表示退货，但此时语义也不完全等价于减法）。  
- **代码质量：** ⚠️ 题目明确要求“Define any you think the class should include.”（定义你认为该类应该包含的运算符），但你只列出了运算符名称，没有给出任何函数声明或定义，这是一个重大疏漏。  
- **逻辑完整：** ⚠️ 答案不完整，缺少对运算符的具体定义，也没有解释如何实现它们。  
- **表达清晰：** ✅ 语言表达基本通顺，但缺乏对为何不支持其他运算符的深入分析。

**正确答案：**  
通常 `Sales_data` 应该支持 `+` 和 `+=` 运算符，用于将两个销售记录合并（相加销量和收入）。示例定义如下：

```cpp
class Sales_data {
public:
    // 复合赋值运算符：通常作为成员函数
    Sales_data& operator+=(const Sales_data& rhs);
    // 其他成员...
};

// 加法运算符：通常作为非成员函数以支持对称转换
Sales_data operator+(const Sales_data& lhs, const Sales_data& rhs) {
    Sales_data sum = lhs;   // 拷贝左侧对象
    sum += rhs;             // 利用复合赋值实现加法
    return sum;
}
```

对于 `-` 和 `-=`，一般不需要支持，除非业务逻辑确实需要（例如退货），但此时也应谨慎设计，确保语义清晰。

### 题目 14.14

**原题：**  
Exercise 14.14: Why do you think it is more efficient to define operator+ to call operator+= rather than the other way around?

**你的答案：**  
首先我们分析一下+=需要的操作有哪些？  
1. 首先本身并不会拷贝创建类  
2. 只进行对成员的操作  

然而+的语意是需要创建出一个新的值， 并且里面的value就是rhs和lhs的和的value, 也就是说相比+=操作来说多了一个创建临时变量其他则是一样，这样来说，其实+操作符积极不需要  

**批改结果：** ⭐⭐⭐⭐ (4/5)  
- **概念理解：** ✅ 你抓住了效率差异的核心：`+=` 直接修改左操作数，不产生新对象；而 `+` 必须返回一个新对象，因此会多一次拷贝（或移动）。你正确指出 `+=` “本身并不会拷贝创建类”，`+` “多了一个创建临时变量”，这很好地解释了为什么用 `+=` 实现 `+` 更高效。  
- **代码质量：** ✅ 虽然没有代码，但分析的是效率原理，不需要代码。  
- **逻辑完整：** ✅ 逻辑基本正确，但可以补充如果反过来用 `+` 实现 `+=` 会导致的额外开销（如临时对象的构造和赋值）。  
- **表达清晰：** ⚠️ 表达稍显模糊，最后一句“其实+操作符积极不需要”可能想表达“因此用 `+=` 实现 `+` 是高效的”，但措辞不够准确，容易引起误解。

**正确答案：**  
定义 `operator+` 调用 `operator+=` 更高效，因为 `operator+=` 直接操作左操作数，无需创建新对象；而 `operator+` 必须返回一个新对象，如果反过来用 `operator+` 实现 `operator+=`，则需要先创建一个临时对象（通过 `+`），再将其赋值给左操作数，这会导致一次额外的拷贝构造和一次赋值操作，降低了效率。典型实现如下：

```cpp
// 高效版本：+ 调用 +=
Sales_data operator+(const Sales_data& lhs, const Sales_data& rhs) {
    Sales_data sum = lhs;   // 一次拷贝
    sum += rhs;             // 直接相加
    return sum;             // 可能发生拷贝或移动
}

// 低效版本：+= 调用 +（不推荐）
Sales_data& Sales_data::operator+=(const Sales_data& rhs) {
    *this = *this + rhs;    // 先通过 + 生成临时对象，再赋值，导致额外开销
    return *this;
}
```
