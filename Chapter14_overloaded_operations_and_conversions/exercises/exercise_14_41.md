### Exercise 14.41

**原题：**  
Why do you suppose the new standard added lambdas? Explain when you would use a lambda and when you would write a class instead.

**你的答案：**  
```
这两种callable object其实再用途方面有非常大的重叠， 但是还可以分情况：
当我们遇到要传入的是非常简短的函数逻辑的时候， 那我们使用lambda
当相对复杂的时候， 我们使用函数对象
当我们要多次，不同再不同地方使用， 调用的时候， 我们使用函数对象
当我们只再一个地方调用，使用那么使用lambda 
当我们重载了多个函数，无法存入到function模版当中的时候也可以使用lambda来解析包裹然后解决问题
```

**批改：** ⭐⭐⭐⭐ (4/5) 优秀答案！你对lambda和函数对象的核心区别有清晰的理解，并给出了实用的选择建议。语言表达流畅，抓住了主要场景。

**详细分析：**
- ✅ **核心概念正确**：你准确指出了lambda适用于简短的、一次性的函数逻辑，而函数对象适合复杂、多处复用的场景。
- ✅ **实用性思考**：你提到了“重载函数无法存入function模板时用lambda包裹”，这体现了对实际编程中类型擦除和可调用对象包装的理解。
- ⚠️ **表述可更严谨**：个别用词略有口语化，如“再用途方面有非常大的重叠”（应为“在用途方面有非常大的重叠”）、“不同再不同地方”（应为“在不同地方”）。
- ⚠️ **可补充关键点**：lambda的引入主要为了**简化编写**和**支持捕获局部变量**，这一点可以明确提及；函数对象的优势还包括**可以携带更多状态、支持名称复用、封装私有数据**等，这些是选择类而非lambda的重要理由。

**正确答案参考：**  
```cpp
// Lambda 示例：简短、一次性、捕获局部变量
int x = 10;
auto addX = [x](int y) { return x + y; };

// 函数对象类示例：复杂逻辑、多处复用、携带更多状态
class Adder {
    int base;
public:
    Adder(int b) : base(b) {}
    int operator()(int y) const { return base + y; }
    // 可以添加更多成员函数和数据
};

// 何时使用 lambda：
// - 逻辑简单（一两行）
// - 只在局部使用一次
// - 需要捕获局部作用域中的变量
// - 希望代码更紧凑、可读性更好

// 何时使用函数对象类：
// - 逻辑复杂，需要多个成员函数或数据
// - 需要在多处重用该可调用对象
// - 需要名称来表明意图
// - 需要访问类的私有成员（定义为内部类或友元）
// - 需要支持多态（通过基类指针调用）
```
