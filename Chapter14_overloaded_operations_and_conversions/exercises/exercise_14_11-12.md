### Exercise 14.11 Sales_data 输入操作符问题分析

**原题：**  
Exercise 14.11: What, if anything, is wrong with the following `Sales_data` input operator? What would happen if we gave this operator the data in the previous exercise?

```cpp
istream& operator>>(istream& in, Sales_data& s)
{
    double price;
    in >> s.bookNo >> s.units_sold >> price;
    s.revenue = s.units_sold * price;
    return in;
}
```
input: 
```
(a) 0-201-99999-9 10 24.95
(b) 10 24.95 0-210-99999-9
```

**你的答案：**
```
首先并不会报错，但是并不会按照逻辑输出罢了：
ISBN number: 0-201-99999-9 Average price: 24.95 Sold number: 10 
ISBN number: 10 Average price: 0.95 Sold number: 24
```

**批改：** ⭐⭐ (2/5)  
你的答案注意到了输入(b)会产生非预期的输出，这是一个好的观察点。但回答过于简略，没有深入分析操作符的根本问题，也没有解释为什么会出现这样的输出，因此未能完整回答题目的两个问题。

**详细分析：**
- ✅ 你正确指出了程序不会报错（编译通过，运行时也不会崩溃），并给出了具体的输出示例，说明你理解了输入(b)会导致数据错位。
- ⚠️ 你没有分析操作符的缺陷：它没有检查输入操作是否成功，也没有在输入失败时正确处理流状态（如设置`failbit`），这会导致对象处于不一致的状态，甚至影响后续操作。
- ⚠️ 对于输入(b)，你只描述了结果，但没有解释为什么`bookNo`会变成`"10"`，`units_sold`变成`24`，`price`变成`0.95`。这是因为输入格式与预期不符，但操作符仍然按顺序读取数据，导致数据被错误地赋给了成员变量。
- ❌ 你忽略了操作符的另一个重要问题：当输入失败时（例如读取`price`时遇到非数字），操作符应该将`s`恢复为原状态（或至少保持有效状态），但当前代码没有做任何错误处理，可能导致部分修改的对象被使用。
- ❌ 回答没有涵盖题目要求的“what is wrong”和“what would happen”两个部分，缺少对错误本质的分析。

**正确答案：**  
这个输入操作符存在以下问题：
1. **缺少错误检查**：没有检查输入操作是否成功。如果任何一次读取失败（例如输入格式错误、遇到文件结束符等），流会进入错误状态，但函数仍然执行 `s.revenue = s.units_sold * price;`，此时 `s.units_sold` 和 `price` 的值是未定义的（可能保持原值或不确定），导致对象状态混乱。
2. **没有维护流状态**：当输入失败时，应该设置流的错误状态（如 `failbit`），并可能将对象恢复到输入前的状态（或至少保持有效状态）。但当前代码直接返回 `in`，而流可能已经处于错误状态，调用者需要检查流状态才能知道输入是否成功。
3. **异常安全性**：如果输入部分成功（例如 `bookNo` 和 `units_sold` 成功，但 `price` 失败），对象已被部分修改，这破坏了强异常安全保证。

对于给定的输入：
- **(a) 0-201-99999-9 10 24.95**：输入格式完全匹配，操作符正常执行，`s` 被正确赋值。
- **(b) 10 24.95 0-210-99999-9**：输入顺序与预期不符。第一个读取 `s.bookNo` 时会将字符串 `"10"` 赋给 `bookNo`（因为 `bookNo` 是 `string` 类型，可以接受任何字符串），第二个读取 `s.units_sold` 得到 `24`（从 `24.95` 转换为整数 `24`，发生截断），第三个读取 `price` 得到 `0.95`（从 `0-210-99999-9` 转换，但字符串以数字开头，`operator>>(double)` 会读取 `0`，然后遇到非数字停止，所以得到 `0.0`？等一下，实际上 `0-210-99999-9` 以 `0` 开头，后面是 `-`，所以 `>> double` 会读入 `0`，然后遇到 `-` 停止，所以 `price` 得到 `0.0`，然后后续字符留在流中。但学生给出的输出是 `Average price: 0.95`，这可能有误。我们重新计算：输入(b)是 `"10 24.95 0-210-99999-9"`。读取顺序：
  - `in >> s.bookNo` 读取 `"10"` 到 `bookNo`（字符串，接受任何字符直到空白）。
  - `in >> s.units_sold` 读取下一个整数，流中剩余 `"24.95 0-210-99999-9"`，`>>` 会读取 `24`（遇到 `.` 停止），`units_sold` 变为 `24`。
  - `in >> price` 读取下一个 double，流中剩余 `".95 0-210-99999-9"`，`>>` 会读取 `0.95`（因为 `.95` 是合法的 double 表示），所以 `price` 变为 `0.95`。
  - 最后 `revenue = 24 * 0.95 = 22.8`。
所以对象状态为：`bookNo="10"`, `units_sold=24`, `revenue=22.8`，这与学生的输出 `ISBN number: 10 Average price: 0.95 Sold number: 24` 一致（平均价格 = revenue/units_sold = 22.8/24 = 0.95）。学生给出的输出是正确的，但缺少对过程的解释。
