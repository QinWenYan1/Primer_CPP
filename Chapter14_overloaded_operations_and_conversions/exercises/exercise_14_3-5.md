### Exercise 14.3
**原题：**  
Both string and vector define an overloaded == that can be used to compare objects of those types. Assuming svec1 and svec2 are vectors that hold strings, identify which version of == is applied in each of the following expressions:  
(a) "cobble" == "stone"  
(b) svec1[0] == svec2[0]  
(c) svec1 == svec2  
(d) "svec1[0] == "stone"

**你的答案：**  
(a) "cobble" == "stone" ， 不合适， 这是单独的string的操作范围， 不是svec的操作  
(b) svec1[0] == svec2[0]， 不合适， 这也是单独string的比较，不是svec的操作  
(c) svec1 == svec2， 合适， 属于svec的操作范畴  
(d) "svec1[0] == "stone"， 不合适， 这是单独string的比较，不是svec的操作  

**批改结果：** ⭐⭐ (2/5)  
- **概念理解：** ❌ 对字符串字面量的类型和运算符重载机制理解有偏差。  
- **代码质量：** 不涉及代码，但分析逻辑不够严谨。  
- **逻辑完整：** ⚠️ 部分判断正确（c），但（a）（d）完全错误，（b）表述模糊。  
- **表达清晰：** ✅ 语言基本通顺，但术语使用不够准确（如“string的操作范围”不精确）。

**详细分析：**  
- ✅ (c) 正确识别了 `svec1 == svec2` 调用的是 `vector` 的 `==`。  
- ⚠️ (b) 实际上调用的是 `string` 的 `==`，因为 `svec1[0]` 返回 `string&`。你的回答“不合适”和“单独string的比较”虽然点出了是string，但“不合适”一词容易引起误解（题目只是要求识别哪个版本，并不评价“合适”与否），且未明确说出是 `string` 的 `==`。  
- ❌ (a) 和 (d) 错误。`"cobble"` 和 `"stone"` 是字符串字面量，类型为 `const char[7]` 和 `const char[6]`，在表达式中会退化为 `const char*`，因此比较的是指针地址，**不是** `string` 的 `==`。你误以为是“string的操作范围”。  
- ❌ (d) 中 `"svec1[0] == "stone"` 是两个字符串字面量的比较（注意写法是带引号的 `"svec1[0]"`，不是变量），同样是指针比较，并非 `string` 比较。

**正确答案：**  
```cpp
(a) 内置指针比较（const char* 比较）
(b) string 的 ==
(c) vector<string> 的 ==
(d) 内置指针比较（const char* 比较）
```

**知识点解析：**  
- **字符串字面量**的类型是 `const char[N]`，在表达式中通常隐式转换为 `const char*`，其 `==` 比较的是地址，而非内容。若要比较内容，需使用 `std::string` 对象或 `strcmp`。  
- `vector` 的 `==` 会逐个比较元素，要求元素类型支持 `==`（这里 `string` 支持）。  
- 下标运算 `svec1[0]` 返回的是 `string` 对象的引用，因此 `svec1[0] == svec2[0]` 调用 `string` 的 `==`。

### Exercise 14.4
**原题：**  
Explain how to decide whether the following should be class members:  
(a) %  
(b) %=  
(c) ++  
(d) ->  
(e) <<  
(f) &&  
(g) ==  
(h) ()

**你的答案：**  
(a) 这个值会改变类的state因此应该定义为成员  
(b) 这个复合操作符也会改变对象的state， 应该定义为成员  
(c) ++ 同样的道理还是应该定义为成员  
(d) -> 成员访问符必须定义为成员  
(e) << 成员或者非成员都可以， 没有改变对象的状态  
(f) && 非成员， symmetric operator  
(g) == 非成员， 因为时symmetric operator  
(h) () 必须是成员， 这个时函数调用操作符号

**批改结果：** ⭐⭐⭐ (3/5)  
- **概念理解：** ⚠️ (a)和(e)错误，其余正确，但部分理由不够严谨。  
- **代码质量：** 不涉及代码。  
- **逻辑完整：** ✅ 大部分判断正确。  
- **表达清晰：** ✅ 基本清晰，但有错别字（“时”应为“是”）。

**详细分析：**  
- ✅ (b) `%=` 正确：复合赋值通常修改左操作数，应定义为成员。  
- ✅ (c) `++` 正确：自增运算符修改对象状态，应定义为成员（注意前缀和后缀形式都通常为成员）。  
- ✅ (d) `->` 正确：必须为成员。  
- ✅ (f) `&&` 正确：对称二元运算符，通常为非成员（但注意重载 `&&` 会失去短路求值，一般不建议重载）。  
- ✅ (g) `==` 正确：对称性，通常为非成员。  
- ✅ (h) `()` 正确：函数调用运算符必须为成员。  
- ⚠️ (a) `%` 错误：`%` 是算术运算符，不修改操作数，通常应定义为**非成员**以支持对称性（如 `a % b` 和 `b % a` 都可调用）。你说“改变类的state”是不正确的。  
- ⚠️ (e) `<<` 错误：输出运算符通常必须定义为**非成员**，因为左操作数是 `ostream&`，不是该类对象。如果定义为成员，则使用时必须写成 `object << cout`，不符合常规。你说“成员或者非成员都可以”是不准确的，虽然技术上可以定义为成员，但实践中几乎总是非成员友元。

**正确答案：**  
```cpp
(a) 非成员（算术运算符，不修改对象，需要对称性）
(b) 成员（复合赋值，修改左操作数）
(c) 成员（自增修改对象状态）
(d) 成员（箭头运算符必须是成员）
(e) 非成员（左操作数是ostream，不是该类对象）
(f) 非成员（对称二元运算符，但通常不建议重载）
(g) 非成员（对称性）
(h) 成员（函数调用运算符只能是成员）
```
