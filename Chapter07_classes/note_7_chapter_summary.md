# 📘 第七章 类（Class）

> 来源说明：C++ Primer 第七章 | 本节涵盖：类的定义、构造函数、成员函数、访问控制、封装等核心概念
---

### 📚 C++ 类核心概念知识表

| 类别 | 概念 | 定义 | 关键点 |
|------|------|------|--------|
| **🔧 类基础** | **class** | C++ 中用于定义自定义抽象数据类型的机制。 | - 可包含数据、函数或类型成员<br>- 定义新类型和新作用域 |
|  | **struct** | 用于定义类的关键字；默认成员为 public。 | - 与 class 类似，默认访问权限不同<br>- 适用于简单数据聚合 |
|  | **class scope** | 每个类定义的作用域。 | - 成员函数可在类体内使用后出现的名称<br>- 作用域规则比普通作用域复杂 |
|  | **class declaration** | class 或 struct 关键字后跟类名和分号。 | - 若未定义，则为**不完全类型**<br>- 可用于前向声明 |
|  | **incomplete type** | 已声明但未定义的类型。 | - 不能用于定义变量或类成员<br>- 可定义指向不完全类型的指针或引用 |
| **🏗️ 构造函数** | **constructor** | 用于初始化对象的特殊成员函数。 | - 每个构造函数应为数据成员提供明确初始值<br>- 与类同名，无返回类型 |
|  | **default constructor** | 未提供初始化值时使用的构造函数。 | - 可由编译器合成（synthesized）<br>- 若无自定义构造函数，编译器自动生成 |
|  | **synthesized default constructor** | 编译器为未定义构造函数的类生成的默认构造函数。 | - 使用类内初始值（若有）初始化成员<br>- 否则默认初始化成员 |
|  | **= default** | 在类内部默认构造函数声明后使用，指示编译器生成该构造函数。 | - 即使类中有其他构造函数也可使用<br>- 显式要求编译器生成默认版本 |
|  | **constructor initializer list** | 指定类数据成员初始值的列表。 | - 在构造函数体执行前初始化成员<br>- 未列出的成员将默认初始化 |
|  | **converting constructor** | 可被单个参数调用的非 explicit 构造函数。 | - 支持从参数类型到类类型的**隐式转换**<br>- 可能导致意外类型转换 |
|  | **explicit constructor** | 可被单参数调用但不能用于隐式转换的构造函数。 | - 使用 `explicit` 关键字声明<br>- 防止编译器自动执行不期望的转换 |
|  | **delegating constructor** | 构造函数初始化列表中调用同类其他构造函数的构造函数。 | - 减少代码重复<br>- 提高构造函数复用性 |
| **👥 成员与封装** | **member function** | 类的函数成员。 | - 非静态成员函数通过 `this` 指针绑定到对象<br>- 可重载，`this` 参与匹配 |
|  | **const member function** | 不修改对象普通数据成员的成员函数。 | - `this` 指针为指向 const 的指针<br>- 可基于 const 重载 |
|  | **this pointer** | 隐式传递给每个非静态成员函数的额外参数。 | - 指向调用该函数的对象<br>- 在成员函数中隐式使用 |
|  | **mutable data member** | 即使在 const 对象中也不会变为 const 的数据成员。 | - 可在 const 成员函数中修改<br>- 用于表示对象内部状态（如缓存、计数器） |
|  | **private members** | 在 private 访问说明符后定义的成员。 | - 仅可由类的成员和友元访问<br>- 通常用于封装实现细节 |
|  | **public members** | 在 public 访问说明符后定义的成员。 | - 任何用户都可访问<br>- 通常定义类的接口 |
|  | **access specifier** | public 和 private 关键字，控制成员访问权限。 | - 可多次出现在类中<br>- 每个说明符影响其后成员直至下一个说明符 |
|  | **friend** | 允许类授予对非公有成员访问权限的机制。 | - 友元可以是类或函数<br>- 友元拥有与成员相同的访问权限 |
| **📦 抽象与封装** | **abstract data type (ADT)** | 封装（隐藏）其实现的数据结构。 | - 强调接口而非实现<br>- 提高代码模块化和可维护性 |
|  | **data abstraction** | 关注类型接口的编程技术。 | - 忽略类型表示细节，关注其操作<br>- 面向对象和泛型编程的基础 |
|  | **encapsulation** | 实现与接口的分离。 | - 在 C++ 中通过将实现置于 private 部分实现<br>- 隐藏实现细节，增强安全性 |
|  | **interface** | 类型支持的（公共）操作集合。 | - 通常不包括数据成员<br>- 定义类的外部行为 |
|  | **implementation** | 类的（通常为私有）成员，定义数据和不供外部使用的操作。 | - 用户代码不应直接访问<br>- 封装在类内部，支持数据抽象 |
|  | **aggregate class** | 仅包含公有数据成员、无类内初始化器或构造函数的类。 | - 可使用花括号初始化列表初始化成员<br>- 适用于简单数据结构 |
