## 📊 C++ 继承与多态 — 结构化知识表

| 类别 | 概念 | 定义 | 关键点 |
|------|------|------|--------|
| **🎯 核心范式** | 继承 (Inheritance) | 定义新类（派生类）以现有类（基类）为基础的编程技术，派生类继承基类成员 | • 实现代码复用和层次结构<br>• 支持扩展（添加新成员）和特化（覆盖行为） |
| | 面向对象编程 (Object-oriented programming) | 使用数据抽象、继承和动态绑定的编程方法 | 三大支柱：封装、继承、多态 |
| | 重构 (Refactoring) | 重新设计程序以收集相关部分到单一抽象，用新抽象替代原始代码 | • 通常将成员移至继承层次的最高公共点<br>• 目的：避免代码重复 |
| **🏗️ 类层次结构** | 基类 (Base class) | 其他类从中继承的类，其成员成为派生类的成员 | 作为派生类的基础和接口 |
| | 派生类 (Derived class) | 从另一个类继承的类，可覆盖虚函数并定义新成员 | • 作用域嵌套在基类作用域内<br>• 可直接使用基类成员 |
| | 直接基类 (Direct base class) | 在派生类的派生列表中明确指定的基类 | • 本身也可以是派生类<br>• 区别于间接基类 |
| | 间接基类 (Indirect base class) | 不出现在派生类派生列表中，但通过直接基类继承而来的基类 | 继承链中的"祖父类"及以上 |
| | 抽象基类 (Abstract base class) | 含有一个或多个纯虚函数的类 | • 不能创建该类型的对象<br>• 用于定义接口规范 |
| **📋 类定义语法** | 类派生列表 (Class derivation list) | 基类列表，每个基类可带有可选的访问级别 | • 默认访问：`struct`为public，`class`为private<br>• 多重继承时列出多个基类 |
| **🔄 类型与转换** | 静态类型 (Static type) | 变量定义时的类型或表达式产生的类型，编译时已知 | 与 dynamic type 相对 |
| | 动态类型 (Dynamic type) | 对象运行时的实际类型 | • 指针/引用的静态类型可能与动态类型不同<br>• 基类指针可指向派生类对象（动态类型为派生类） |
| | 派生到基类转换 (Derived-to-base conversion) | 派生类对象隐式转换为基类引用，或派生类指针隐式转换为基类指针 | 基于"派生类包含基类子对象"的事实 |
| | 对象切片 (Sliced down) | 用派生类对象初始化或赋值基类对象时，派生部分被"切掉"，仅保留基类部分 | • 导致数据丢失<br>• 避免：使用指针或引用而非对象值 |
| **⚡ 多态机制** | 动态绑定 (Dynamic binding / Run-time binding) | 延迟到运行时选择要执行的函数版本 | • 仅适用于 `virtual` 函数<br>• 必须通过引用或指针调用<br>• 根据对象的动态类型解析 |
| | 虚函数 (Virtual function) | 定义类型特定行为的成员函数 | • 通过引用/指针调用时在运行时解析<br>• 实现多态的基础机制 |
| | 纯虚函数 (Pure virtual) | 在类头中使用 `= 0` 声明的虚函数 | • 无需（但可以有）定义<br>• 含纯虚函数的类为抽象类<br>• 派生类必须实现，否则也为抽象类 |
| | 覆盖 (Override) | 派生类中定义的与基类虚函数参数列表相同的虚函数 | 替代基类的实现，实现运行时多态 |
| | 多态 (Polymorphism) | 根据引用或指针的动态类型获取类型特定行为的能力 | OOP 核心特性，通过虚函数实现 |
| **🔐 访问控制** | 可访问性 (Accessible) | 可通过派生对象使用的基类成员 | • 取决于派生列表中的访问限定符<br>• 取决于基类中成员的访问级别 |
| | 保护访问限定符 (Protected access specifier) | `protected` 关键字后定义的成员 | • 可被派生类的成员和友元访问<br>• 对类的普通用户不可见 |
| | 公有继承 (Public inheritance) | 基类的 `public` 接口成为派生类的 `public` 接口 | 表达"is-a"关系，最常用 |
| | 保护继承 (Protected inheritance) | 基类的 `protected` 和 `public` 成员成为派生类的 `protected` 成员 | 对外隐藏基类接口，对派生类可见 |
| | 私有继承 (Private inheritance) | 基类的 `public` 和 `protected` 成员成为派生类的 `private` 成员 | 表达"is-implemented-in-terms-of"关系 |

## 📝 重要机制补充

| 机制 | 规则 | 注意事项 |
|------|------|----------|
| **对象构造** | 先构造基类部分，再构造派生部分 | 按派生列表中的顺序构造基类 |
| **对象析构** | 先析构派生部分，再析构基类部分 | 与构造顺序相反 |
| **虚析构函数** | 基类通常应定义虚析构函数 | • 即使类本身不需要析构函数<br>• 必须通过基类指针删除派生对象时必需 |
