### Exercise 15.1: 什么是虚成员？

**原题：** What is a virtual member?

**你的答案：**
> 虚成员就是类中需要被派生类继承并override的成员来定义接口的表现，虚成员在基类中可以不提供实际实际的实现，可以在派生类中去实现该成员。一般在派生类中的实现可以带virtual关键字或者不带， 但是在新标准中时推荐使用`override`来让编译器知道我们在这里实现了对虚函数的实现。当然虚函数的定义可以是在基类， 也可以在派生类， 这个都是自由的。另外如果派生类如果没有提供virtual的实现，那么编译器会去基类中查找该virtual函数的实现并调用。 
> 
> 这些行为都是运行时的动作

**批改结果：** ⭐⭐⭐ (3/5)

**详细分析：**
- ✅ 正确指出虚成员支持派生类进行override
- ✅ 正确说明`override`关键字的使用（C++11最佳实践）
- ✅ 正确指出如果派生类未覆盖，会调用基类版本
- ✅ 正确识别动态绑定（运行时行为）
- ⚠️ **概念混淆**：将"虚函数"与"纯虚函数"混为一谈。只有纯虚函数（`= 0`）才必须在基类不提供实现，普通虚函数在基类必须有实现
- ⚠️ **表述不清**："虚成员在基类中可以不提供实际的实现"这句话容易误导，应明确区分纯虚函数和普通虚函数
- ⚠️ **术语问题**："定义可以是在基类，也可以在派生类"表述模糊，虚函数声明必须在基类，定义可以在类内或类外

**正确答案：**
```cpp
// 虚成员（主要指虚函数）是声明时带有 virtual 关键字的类成员函数
// 它支持动态绑定（运行时多态），允许派生类重写(override)基类实现

class Base {
public:
    // 普通虚函数：基类提供默认实现，派生类可选择性覆盖
    virtual void func() { /* 基类实现 */ }
    
    // 纯虚函数：基类不实现（=0），派生类必须覆盖，使类成为抽象类
    virtual void pure() = 0;  
};

class Derived : public Base {
public:
    void func() override { /* 覆盖基类实现 */ }  // 覆盖普通虚函数
    void pure() override { /* 必须实现 */ }      // 实现纯虚函数
};
```

**关键知识点：**
- **虚函数 (Virtual Function)**：基类使用`virtual`声明，支持动态绑定。基类通常提供默认实现，派生类可选择覆盖。使用基类指针/引用调用时，实际调用哪个版本由对象类型在运行时决定
- **纯虚函数 (Pure Virtual Function)**：声明为`= 0`，基类不提供实现，包含纯虚函数的类是抽象类，不能实例化，派生类必须实现所有纯虚函数才能实例化
- **动态绑定 vs 静态绑定**：虚函数调用在运行时解析（通过虚函数表vptr/vtable），普通成员函数调用在编译时解析

### Exercise 15.2: `protected`与`private`访问说明符的区别？

**原题：** How does the `protected` access specifier differ from `private`?

**你的答案：**
> 有的`proteceted`标识表示该成员可以被派生类访问， 而非派生类是无法访问的。对于`private`来说无论访问成员的类是不是派生类都会被拒绝。 因此两者的区别是显而易见的，就是他们的访问scope是不一样的。

**批改结果：** ⭐⭐⭐⭐ (4/5)


**标准表述：**
```cpp
class Base {
private:
    int priv;      // 仅类自身可访问，友元可访问，派生类不可访问
protected:
    int prot;      // 类自身、友元、派生类可访问，类外不可访问
public:
    int pub;       // 任何地方都可访问
};

class Derived : public Base {
    void func() {
        // priv = 1;  // 错误：无法访问基类private成员
        prot = 2;     // 正确：可以访问基类protected成员
        pub = 3;      // 正确：可以访问基类public成员
    }
};
```
