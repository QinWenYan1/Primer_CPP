### Exercise 11.5: map和set的区别

**原题：** Explain the difference between a map and a set. When might you use one or the other?

**你的答案：**
```
map的原书是由pair类组成，其中pair类又由first和second成员组成
构成了键值对,key-value
然而set则是由单key构成而没有value
```

**批改结果：** ⭐⭐⭐ (3/5)
- **概念理解：** 3/4 - 正确指出了基本结构区别
- **代码质量：** N/A (文字题)
- **逻辑完整：** 1/2 - 缺失了使用场景的说明
- **表达清晰：** 1/2 - 表述基本清晰，但"原书"可能是笔误

**详细分析：**
- ✅ 正确指出了map由键值对(pair)组成
- ✅ 正确指出了set只包含键(元素)而没有值
- ⚠️ 未回答题目要求的"何时使用"部分
- ⚠️ 表述中的"原书"可能是"元素"的笔误

**正确答案：**
```markdown
map和set都是C++标准库中的关联容器，主要区别如下：

1. **结构差异**：
   - map存储键值对(key-value pairs)，每个元素是pair<const Key, Value>
   - set只存储键(key)，每个元素就是键本身

2. **底层实现**：
   - 两者通常都基于红黑树实现(有序版本)，或哈希表实现(unordered版本)
   - 都保证键的唯一性

3. **使用场景**：
   - 使用map当需要键值映射关系时，如字典、电话簿(名字→号码)
   - 使用set当只需检查元素是否存在或去重时，如黑名单、唯一值集合

4. **性能特征**：
   - 两者查找、插入、删除操作的时间复杂度都是O(log n)(有序版本)或平均O(1)(无序版本)
   - set通常比map占用更少内存，因为没有value部分
```

### Exercise 11.6: `set`和`list`的区别

**原题：** Explain the difference between a set and a list. When might you use one or the other?

**你的答案：**
```
set为无序，list为有序，本质上的区别造就了两者在operation上也有差别，
比如front, back等位置相关操作都不可以使用，使用数量或者一个元素值指定的构造函数set也是不支持的
```

**批改结果：** ⭐⭐ (2/5)
- **概念理解：** 1/4 - 对set的有序性理解有误
- **代码质量：** N/A (文字题)
- **逻辑完整：** 1/2 - 未完整说明使用场景
- **表达清晰：** 1/2 - 表述有一定逻辑性

**详细分析：**
- ❌ **概念错误**：std::set默认是有序的(基于红黑树)，std::unordered_set才是无序的
- ✅ 正确指出了一些操作上的差异
- ⚠️ 未完整回答"何时使用"部分
- ⚠️ 对构造函数的描述不够准确

**正确答案：**
```markdown
set和list是两种完全不同的容器：

1. **基本性质**：
   - set：关联容器，元素自动排序且唯一，基于红黑树实现
   - list：顺序容器，元素按插入顺序排列，基于双向链表实现

2. **主要区别**：
   - **排序与唯一性**：set元素自动排序且唯一；list保持插入顺序，允许重复
   - **数据结构**：set通常基于平衡树；list基于双向链表
   - **访问方式**：set支持高效查找(O(log n))；list支持快速插入删除(O(1))
   - **迭代器**：set提供双向迭代器但元素只读；list提供双向迭代器可修改元素
   - **位置操作**：list支持front()、back()、push_front()等；set不支持这些

3. **使用场景**：
   - 使用set当需要：快速查找、自动去重、自动排序
     * 如：维护唯一ID集合、字典序存储单词
   - 使用list当需要：频繁在任意位置插入删除、需要稳定迭代器
     * 如：LRU缓存实现、需要频繁重排序的列表

4. **构造函数**：
   - set支持从范围、初始化列表构造，也支持比较器参数
   - list支持从范围、初始化列表构造，也支持指定大小和初始值
```
