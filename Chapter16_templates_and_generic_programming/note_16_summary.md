## 📊 C++ Templates 章节知识表

**章节来源**：C++ Templates 章节总结与术语定义  
**核心主题**：模板机制、实例化、特化与参数管理

| 类别 | 概念 | 定义 | 关键点 |
|------|------|------|--------|
| 🧱 **模板基础** | template | 编译器用于生成特定类类型或函数的蓝图 | • 写一次，编译器根据使用类型自动实例化 • 是C++的标志性特性 • 标准库算法（函数模板）和容器（类模板）的基础 |
| 🧱 **模板基础** | instantiate | 编译器使用实际模板实参生成模板特定实例的过程 | • 函数自动根据调用实参实例化 • 类模板必须显式提供模板实参 |
| 🧱 **模板基础** | instantiation | 编译器从模板生成的类或函数 | • 是模板实例化的结果 |
| 📐 **模板类型** | class template | 用于实例化特定类的定义 | • 使用`template`关键字 + `<>`包裹的模板形参列表 + 类定义 |
| 📐 **模板类型** | function template | 用于实例化特定函数的定义 | • 使用`template`关键字 + `<>`包裹的模板形参列表 + 函数定义 |
| 📐 **模板类型** | member template | 作为模板的成员函数 | • **不能是虚函数** |
| 📐 **模板类型** | variadic template | 接受可变数量模板实参的模板 | • 使用省略号`...`指定（如`class...`、`typename...`） • C++标准最新特性的重要组成 |
| 🔧 **模板参数** | template parameter | 模板形参列表中指定的名称，可在模板定义内部使用 | • 可以是**类型形参**或**非类型形参** • 使用类模板时必须显式提供实参 • 函数模板可通过调用实参推断 |
| 🔧 **模板参数** | type parameter | 用于表示类型的模板形参名称 | • 使用`typename`或`class`关键字后指定 |
| 🔧 **模板参数** | nontype parameter | 表示值的模板形参 | • 模板实参必须是**常量表达式** |
| 🔧 **模板参数** | template parameter list | 用于模板定义或声明的形参列表（逗号分隔） | • 每个形参可以是类型或非类型参数 |
| 📦 **参数包** | parameter pack | 表示零个或多个参数的模板或函数形参 | • 可变参数模板的核心机制 |
| 📦 **参数包** | template parameter pack | 表示零个或多个模板形参的参数包 | • 用于可变参数模板类/函数 |
| 📦 **参数包** | function parameter pack | 表示零个或多个函数形参的参数包 | • 传递参数给对象构造函数 |
| 📦 **参数包** | pack expansion | 用对应元素列表替换参数包的过程 | • 实现可变参数展开的关键机制 |
| 📦 **参数包** | pattern | 定义展开参数包中每个元素的形式 | • 控制包展开的具体方式 |
| ⚙️ **实例化控制** | template argument | 用于实例化模板形参的类型或值 | • 显式提供或编译器推断 |
| ⚙️ **实例化控制** | explicit template argument | 用户在调用函数或定义模板类类型时提供的模板实参 | • 在模板名后的尖括号`<>`中提供 • 可固定一个或多个模板形参的类型/值 • 正常转换适用于带显式实参的形参 |
| ⚙️ **实例化控制** | default template arguments | 用户未提供对应实参时模板使用的类型或值 | • 提供默认值的机制 |
| ⚙️ **实例化控制** | explicit instantiation | 为所有模板形参提供显式实参的声明 | • 用于指导实例化过程 • `extern`声明表示模板不会实例化，否则用指定实参实例化 • 每个`extern`声明必须在程序某处有非`extern`显式实例化 |
| ⚙️ **实例化控制** | template argument deduction | 编译器确定实例化哪个函数模板的过程 | • 检查使用模板形参指定的实参类型 • 自动实例化绑定这些类型/值的函数版本 |
| 🎯 **特化机制** | template specialization | 用户提供的模板实例化，将一个或多个模板形参绑定到指定类型或值 | • 适用于无法（或不想）使用通用模板定义的类型 • 必须在基模板声明之后、使用特化实参之前出现 • 函数模板每个形参必须完全特化 |
| 🎯 **特化机制** | partial specialization | 部分（非全部）模板形参被指定的类模板版本 | • 部分形参未完全指定 • 仅适用于类模板 |
| 🔄 **类型处理** | type transformation | 标准库定义的类模板，将给定模板类型形参转换为相关类型 | • 标准库提供的类型变换工具 |

## 📌 重点概念速查

### 可变参数模板核心机制
```
template <typename... Args>        // template parameter pack
void func(Args... args) {          // function parameter pack
    // pack expansion: args...
}
```

### 显式实例化 vs 显式实参
| 特性 | explicit instantiation | explicit template argument |
|------|------------------------|---------------------------|
| **目的** | 控制实例化时机和位置 | 指定模板形参的具体类型/值 |
| **语法位置** | 声明语句 | 函数调用或类定义时 |
| **extern用法** | `extern template...` 抑制实例化 | 不适用 |

### 模板形参分类
- **类型形参** (`typename T` / `class T`)：代表一个类型
- **非类型形参** (`int N`)：代表一个常量值
- **模板模板形参** (`template<class> class Container`)*：*未在本章中详细定义
