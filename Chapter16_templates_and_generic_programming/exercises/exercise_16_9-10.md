## 🎯 Exercise 16.9: 函数模板与类模板定义

**原题：** What is a function template? What is a class template?

**你的答案：**
> 用于实例化函数的模版就是function template，用于实例化类的模版就是class template。

**批改结果：** ⭐⭐ 2/5

**详细分析：**
- ⚠️ **循环定义问题**：你说"用于实例化函数的就是函数模板"，这等于说"函数模板就是函数模板"，没有解释本质
- ❌ **概念混淆**：模板本身不是"用于实例化"的工具，模板本身就是**蓝图/模具**（blueprint），实例化是编译器根据蓝图生成代码的过程
- ❌ **缺少核心特征**：没有说明模板参数化的是什么（类型？非类型？），也没有说明模板本身不是函数/类

**正确答案：**
```cpp
// 函数模板：参数化的是函数可以接受的不同类型
// 它本身不是函数，而是一个"函数生成器"
template <typename T>
T add(T a, T b) { return a + b; }  // 蓝图：描述"如何将两个同类型值相加"

// 类模板：参数化的是类成员的类型或值
// 它本身不是类，而是一个"类生成器"  
template <typename T>
class Box {
    T value;  // 参数化：value可以是int、string、任何类型
};
```

**关键知识点：**
- **函数模板**：一个独立于具体类型的函数蓝图，描述了函数的**逻辑结构**，但允许操作不同类型的数据。编译器根据实参推导类型并生成具体函数（模板函数）
- **类模板**：一个独立于具体类型的类蓝图，描述了类的**数据结构**，但成员类型可以变化。编译器根据显式指定的模板参数生成具体类（模板类）
- **核心区别**：函数模板通过**实参推导**类型（通常），类模板必须**显式指定**类型参数

## 🎯 Exercise 16.10: 类模板实例化时发生什么

**原题：** What happens when a class template is instantiated?

**你的答案：**
> 仅仅是这个类被初始化了，类型确定下来了而已，成员依然需要在调用的时候或者访问的时候才实例化

**批改结果：** ⭐⭐⭐ 3/5

**详细分析：**
- ✅ **核心概念正确**：你准确抓住了**按需实例化（lazy instantiation）**的关键特性——成员函数确实是在使用时才实例化
- ⚠️ **术语不准确**："初始化"应改为"**实例化**"（instantiation）。初始化（initialization）是运行时的概念，实例化是编译期从模板生成代码的过程
- ⚠️ **不够完整**：数据成员（非静态成员变量）和类型定义是在类实例化时就确定的，只有成员函数是延迟实例化的

**详细解析：**

```cpp
template <typename T>
class MyClass {
    T data;                    // 数据成员：类实例化时确定
    using value_type = T;      // 类型别名：类实例化时确定
    
public:
    void func() { 
        T local = data;        // 成员函数：【使用时】才实例化
    }
    void unused() {
        T::nonexistent();      // 错误代码在这里，但只要不调用就【不报错】！
    }
};

// 类模板实例化
MyClass<int> obj;  // 此时：
// ✅ MyClass<int> 类型已生成
// ✅ data 成员类型确定为 int
// ✅ value_type 确定为 int
// ❌ func() 和 unused() 尚未实例化！

obj.func();        // 此时 func() 才被实例化
// obj.unused();    // 如果调用，才会发现 T::nonexistent() 错误
```

**改进建议：**
1. 区分**数据成员**（类实例化时确定）和**成员函数**（按需实例化）
2. 使用术语"实例化"而非"初始化"
3. 提及**两阶段编译模型**（Two-phase compilation）：模板定义时的语法检查 + 实例化时的语义检查

**记忆技巧：**
> 类模板实例化 = **"搭好骨架，内脏按需长"**  
> 类是骨架（立即确定），成员函数是内脏（用时才长）
