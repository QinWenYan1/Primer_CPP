## 📊 练习批改与分析

### Exercise 16.1 Define instantiation

**原题：** Define instantiation.

**你的答案：**
> 实例化是发生在编译时，当编译器发现有模版类或者模版函数被调用后， 编译器开始检查模版参数是否正确， 检查模版参数类型是否支持这个函数或者类的操作。如果没有问题，编译器会使用这个模版参数和模版类或者模版函数来生成最终的代码， 这个就叫做实例化

**批改：** ⭐⭐⭐ (3/5) 概念理解基本正确，但需要更精确和完整的定义

**详细分析：**
- ✅ 正确指出实例化发生在**编译时**（针对模板而言）
- ✅ 理解到实例化是由**模板参数**触发的代码生成过程
- ✅ 认识到编译器会检查模板参数的合法性
- ⚠️ 缺少**模板参数替换**这一核心概念——实例化的本质是用具体类型/值替换模板形参
- ⚠️ 未区分**隐式实例化**（由编译器自动触发）与**显式实例化**（程序员主动指定）
- ⚠️ 表述略显笼统，"生成最终代码"不够准确，应为"生成具体的类或函数定义"

**正确答案：**
```cpp
// 概念定义（非代码）
/*
Instantiation（实例化）是指：
1. 编译器根据模板（template）和具体模板实参（template arguments）
2. 用实际类型或值替换模板形参（template parameters）
3. 生成具体类或函数定义的过程

关键要点：
- 隐式实例化：当模板被使用时自动发生
- 显式实例化：使用 template class ClassName<Type>; 语法主动触发
- 结果是从模板生成具体的、完整的类或函数，不再是"模板"
*/
```
